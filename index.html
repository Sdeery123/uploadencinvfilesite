Frequently Asked Questions

What is BASE64 encoding and why is it needed?

BASE64 encoding converts binary data into a text format that can be safely transmitted over systems that handle text only. Invoice Cloud requires BIF files to be BASE64 encoded to ensure reliable transmission across different systems.

Can I save the encoded file for future use?

Yes, you can click the 'Save Encoded File' button to save the BASE64 encoded version of your BIF file. This can be useful for debugging or for uploading through other means.

How do I know if my upload was successful?

A successful upload will display a green results panel with a success message, file name, and File GUID. This information will also be logged in the Operation Log section of the application.

Can I use the same BIF file format for all billing types?

Yes, the BIF format is standardized across Invoice Cloud, but different billing types may require specific fields to be included. Consult the Invoice Cloud documentation for your specific industry requirements.

Are my credentials stored securely?

Yes, credentials are stored locally on your computer in an encrypted format. You can also choose not to save credentials at all if your security policies require it.
