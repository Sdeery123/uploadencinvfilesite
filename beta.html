Database Connection Setup

BIF Uploader Beta 3.3.0 supports multiple database engines. Here's how to set up your database connection:

Connection String Examples

Server=myServerAddress;Database=BIFUploader;User Id=myUsername;Password=myPassword;

For SQL Server, you'll need to provide the server address, database name, and credentials. If using Windows Authentication, you can use:

Server=myServerAddress;Database=BIFUploader;Trusted_Connection=True;
Data Source=C:\Path\To\BIFUploader.db;

SQLite is file-based and doesn't require a separate server. Simply specify the path to the database file.

Server=myServerAddress;Database=BIFUploader;Uid=myUsername;Pwd=myPassword;

MySQL connections require the server address, database name, and credentials.

Host=myServerAddress;Database=BIFUploader;Username=myUsername;Password=myPassword;

PostgreSQL connections follow a similar pattern but with different parameter names.

Configuration Steps

  1. Open BIF Uploader Beta and navigate to Settings > Database
  2. Select your database engine from the dropdown
  3. Enter your connection details or connection string
  4. Click "Test Connection" to verify connectivity
  5. Click "Save" to store your configuration
Tip

For development and testing, SQLite provides the simplest setup with no additional server requirements. For production use, we recommend SQL Server or PostgreSQL for their robust features and performance.

Advanced Synchronization Technology

Bidirectional Sync

BIF Uploader Beta 3.3.0 features a sophisticated bidirectional synchronization system that keeps your local SQL database and Invoice Cloud perfectly in sync.

The Balance Conflict Resolution Dialog helps you make informed decisions when discrepancies are detected.

Smart Balance Conflict Resolution

When the system detects differences between local and cloud invoice data, our intelligent conflict resolution system analyzes the nature of the discrepancy and makes smart recommendations:

  • Cloud Balance Lower: If Invoice Cloud shows a lower balance, the system recommends using cloud data, as this typically indicates a payment was made through Invoice Cloud.
  • Local Balance Lower: If your local database shows a lower balance, the system recommends using local data, which might indicate an adjustment that should be pushed to Invoice Cloud.
  • User Control: You can always override recommendations and choose to use local data, cloud data, or skip synchronization entirely for any invoice.
  • Automatic Mode: Set your preferred conflict resolution strategy to be applied automatically without interruption.

Missing Invoice Detection & Upload

One of the most powerful features of the new sync engine is its ability to detect invoices that exist in your local database but not in Invoice Cloud. When this happens:

  • The system automatically identifies invoices that need to be uploaded to Invoice Cloud
  • Invoices are prepared with the correct format and encoding
  • A secure upload process pushes the invoices to Invoice Cloud
  • The system verifies successful upload and updates the local database with confirmation
  • All activity is logged with detailed tracking for audit purposes
Pro Tip

This feature is particularly valuable for organizations that create invoices offline or import them from other systems. You can simply add invoices to your local database, and the sync process will automatically upload them to Invoice Cloud without requiring manual BIF file creation.
