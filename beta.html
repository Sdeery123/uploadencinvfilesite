REST API Key Setup

InvoiceSync Suite Beta 4.0 introduces direct REST API integration with Invoice Cloud, providing a simpler and more efficient upload method.

1

Obtain Your API Key

Contact your Invoice Cloud representative to obtain your REST API key.

2

Configure InvoiceSync Suite

In the application, select "REST API" as your upload format and enter your API key in the credentials section.

3

Save Your Configuration

Check "Save credentials for future use" to securely store your API key for subsequent uploads.

Benefits of REST API

REST API uploads offer several advantages over traditional SOAP methods:

  • Simplified authentication with single API key instead of multiple credentials
  • Improved performance with faster upload times
  • Better error handling and more detailed response information
  • Modern integration standard with better compatibility

Database Connection Setup

InvoiceSync Suite Beta 4.0 supports multiple database engines. Here's how to set up your database connection:

Connection String Examples

Server=myServerAddress;Database=InvoiceSyncSuite;User Id=myUsername;Password=myPassword;

For SQL Server, you'll need to provide the server address, database name, and credentials. If using Windows Authentication, you can use:

Server=myServerAddress;Database=InvoiceSyncSuite;Trusted_Connection=True;
Data Source=C:\Path\To\InvoiceSyncSuite.db;

SQLite is file-based and doesn't require a separate server. Simply specify the path to the database file.

Server=myServerAddress;Database=InvoiceSyncSuite;Uid=myUsername;Pwd=myPassword;

MySQL connections require the server address, database name, and credentials.

Host=myServerAddress;Database=InvoiceSyncSuite;Username=myUsername;Password=myPassword;

PostgreSQL connections follow a similar pattern but with different parameter names.

Key Benefits of REST API Integration

Our REST API implementation offers several important advantages over traditional SOAP-based methods:

  • Simplified Authentication: Use a single API key instead of juggling multiple credentials.
  • Improved Performance: REST API uploads are typically 40-60% faster than equivalent SOAP operations.
  • Better Error Handling: More specific error messages and detailed validation feedback.
  • Enhanced Security: API keys can be rotated and managed with finer access controls.
  • Modern Standard: REST APIs represent the current industry standard for web service integration.

REST vs. SOAP Comparison

Feature REST API SOAP API
Authentication Single API key GUID + Web Service Key
Upload Speed Faster (lightweight) Slower (XML overhead)
Error Details More specific with HTTP status codes Generic SOAP fault messages
Implementation Simpler, more modern More complex protocol
Compatibility Works with all modern platforms May require specific SOAP libraries
Pro Tip

While both REST and SOAP upload options are supported, we recommend using REST API for all new implementations. The REST API provides better performance, simpler authentication, and more detailed error information.
